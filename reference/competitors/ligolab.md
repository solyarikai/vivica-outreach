## Market position

LigoLab is a purpose-built LIS+LIMS+RCM for clinical, anatomic pathology, and outreach labs. They compete on **end-to-end workflow including billing**, which is a strong differentiator vs cloud LIMS that don't touch RCM. On-premise heritage; "hybrid cloud" claims, but real cloud-native is not the norm.

Sales motion: enterprise sales with implementation services. Long sales cycle.

## Where Vivica wins

- **True cloud-native**: Vivica was built cloud, not migrated to it. No on-prem option means no on-prem complexity.
- **Implementation timeline**: LigoLab implementations frequently run 6-12 months. Vivica targets 4-6 weeks.
- **IT burden**: LigoLab on-prem requires server admin, DBA, application admin. Vivica needs none of that.

## Where LigoLab wins

- **End-to-end RCM included** — if a lab specifically wants billing in the same platform, LigoLab is purpose-built for that
- **Established outreach-lab workflows** with multi-client routing
- **Pathologist-friendly AP module**

## Migration angle

Three-phase framework applies. The interesting question for LigoLab migrations: **does the customer want to keep their existing RCM, or did they buy LigoLab partly for the RCM?** If RCM is core to their LigoLab choice, Vivica needs to discuss billing-integration partner options.
