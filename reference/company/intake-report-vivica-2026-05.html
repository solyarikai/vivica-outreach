02
Client · Vivica.us

Cloud-LIMS
с одним сильным якорем

Vivica — комбинированная LIS + LIMS платформа для клинических лабораторий США. Закрывает POL, PSC и Reference-сегменты. Hospital-лабы не их рынок (Epic/Cerner lock-in). Один действующий клиент уровня enterprise.

Положение на старте проекта
CL-01 · 2026-05-11

Acure Reference Lab (NJ, выручка $100M+) — единственный действующий клиент. Vivica работает у них на полном стеке, включая anatomic pathology, есть совместная публикация с KOL.

Что это даёт outreach. Слабая история по количеству, сильнейшая по качеству. Tone — не «лидер рынка», а партнёр-инсайдер с эксклюзивной экспертизой по reference-сегменту. Сильные стороны для писем: цена входа, скорость деплоя, кастомизация под клиента, полный AP-стек. Слабая сторона: отсутствие brand recognition — первый контакт всегда холодный по полной.

Целевые сегменты лабораторий: POL (physician office labs) · PSC (patient service centers / сети забора крови) · Reference Labs (принимают сложные тесты от других лабораторий). Hospital-лаборатории — out of scope.

[LEARN 1]
CMS-first архитектура
Apollo пропускает 30-60% малых лабораторий — нет LinkedIn-присутствия. CMS даёт полный список рынка. Уникальное преимущество vs стандартного outbound-агентства.
design decision · CMS POS Q1 2026
[LEARN 2]
Apollo "verified" — не доверять
При SMTP-double-check через FindyMail 14.4% Apollo-«verified» оказались bounce. Принято: всегда верифицируем дважды до SmartLead. Сохраняет sender reputation.
decisions-log · 2026-05-11
[LEARN 3]
Reveal by id, never name fallback
FindyMail name-search на Apollo-промахах = 0.5% hit rate. Apollo Reveal by id даёт 82.5%. Pattern зафиксирован: people search всегда сохраняет id, fallback на name отключён.
